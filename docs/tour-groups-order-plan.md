# План по изменению отображения и заказу тур-групп

## 1. Изменить отображение списка тур-групп
- Использовать данные из модели `TourGroup`.
- Удалить изображение и описание, чтобы строки были компактнее.
- Отобразить:
  - Название тура.
  - Количество людей, которые заказали тур.
  - Количество свободных мест (рассчитать: всего мест минус заказано).
  - Цена минимальная и максимальная, для максимум человек минимальная цена, для минимального количества людей макчимальная.
  - Кнопку "Заказать" для каждого тура.

## 2. Реализовать кнопку "Заказать"
- При нажатии на кнопку открывается модальное окно.
- В модальном окне форма для ввода контактных данных и сообщения.

## 3. Обработка формы заказа
- При отправке формы:
  - Сохранять сообщение в таблицу `contact_messages` (связать с выбранным туром).
  - Отправлять email с деталями заказа (включая данные о туре и контактные данные пользователя).

## 4. Валидация и UX
- Валидация формы (обязательные поля: имя, email, телефон, сообщение).
- После успешной отправки — показать уведомление об успехе.

## 5. Backend-логика
- Livewire-компонент для обработки формы заказа.
- Модель/метод для сохранения сообщения и отправки email.

## 6. UI/Frontend
- Обновить Blade-шаблон для компактного списка туров.
- Реализовать модальное окно (можно использовать Bootstrap/MDB/Livewire).
- Кнопка "Заказать" для каждого тура.

## 7. Тестирование
- Проверить отображение списка туров.
- Проверить работу модального окна и отправку формы.
- Проверить сохранение сообщения и отправку email.
